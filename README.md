# Бот взаимных подписок

Бот на aiogram 3 для сервиса "накрутки" подписчиков через взаимные подписки между
пользователями. Баллы за выполненное задание сначала замораживаются и зачисляются
на баланс только после повторной проверки (recheck) — это защищает от того, что
человек подписался, получил баллы и сразу отписался.

## Как это устроено

- **Регистрация** (`/start`) — пользователь получает стартовый бонус в баллах.
- **Добавление канала** (`➕ Добавить канал`) — владелец добавляет свой канал.
  Бот должен быть администратором канала, иначе не сможет проверять подписки.
- **Задание** (`📋 Задание`) — пользователю показывается случайный чужой канал.
  После подписки и нажатия «✅ Я подписался» бот проверяет через Bot API,
  что пользователь реально состоит в канале.
- **Заморозка баллов** — при успешной проверке баллы (по умолчанию 0.5) уходят
  в `frozen_balance`, а с владельца канала сразу списывается цена подписки
  (по умолчанию 1.0 балл).
- **Recheck** — раз в 15 минут (настраивается) фоновый планировщик проверяет
  все "замороженные" подписки, которым пора на повторную проверку (по умолчанию
  через 24 часа). Если пользователь всё ещё подписан — баллы зачисляются на
  основной баланс. Если отписался — баллы сгорают, а владельцу канала
  возвращаются его списанные баллы.
- **1 балл = 1 подписчик** — канал показывается в заданиях другим пользователям,
  пока у владельца хватает баллов на балансе.

## Структура проекта

```
subbot/
├── bot.py               # точка входа
├── config.py            # токен и настройки экономики бота
├── db.py                # вся работа с SQLite (aiosqlite)
├── keyboards.py          # reply- и inline-клавиатуры
├── scheduler.py          # фоновая перепроверка подписок (APScheduler)
├── handlers/
│   ├── start.py          # /start, /help
│   ├── balance.py        # /balance
│   ├── channels.py       # добавление/пауза/удаление каналов
│   └── tasks.py          # выдача заданий и проверка подписки
├── requirements.txt
└── .env.example
```

## Установка и запуск

1. Установи Python 3.10+ и зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Получи токен бота у [@BotFather](https://t.me/BotFather) и создай `.env` рядом
   с `bot.py` на основе `.env.example`:

   ```bash
   cp .env.example .env
   ```

   Впиши туда свой токен:

   ```
   BOT_TOKEN=123456789:реальный_токен
   ```

3. Запусти бота:

   ```bash
   python bot.py
   ```

База данных `bot.db` (SQLite) создастся автоматически в той же папке при первом запуске.

## Команды бота

| Команда         | Действие                              |
|-----------------|----------------------------------------|
| `/start`        | Регистрация и открытие главного меню   |
| `/help`         | Как устроен бот                        |
| `/balance`      | Показать баланс (доступно/заморожено)  |
| `/addchannel`   | Добавить свой канал                    |
| `/mychannels`   | Список своих каналов и управление ими  |
| `/task`         | Получить задание на подписку           |
| `/profile`      | Профиль: ID, баланс, кол-во каналов и выполненных заданий |
| `/cancel`       | Отменить текущее действие (например, добавление канала) |

## Важные настройки (`config.py`)

- `START_BONUS` — бонус баллов при регистрации.
- `SUB_REWARD` — сколько баллов получает пользователь за одно выполненное задание.
- `DEFAULT_SUB_PRICE` — сколько баллов списывается с владельца канала за одного подписчика.
- `RECHECK_HOURS` — через сколько часов происходит повторная проверка подписки.
- `SCHEDULER_INTERVAL_MIN` — как часто (в минутах) фоновый планировщик ищет
  подписки, которым пора на перепроверку.

## Деплой на Railway

1. Залей папку `subbot` в свой git-репозиторий (GitHub) и подключи его в Railway
   (New Project → Deploy from GitHub repo). `Procfile` уже есть, Railway сам
   определит Python-проект по `requirements.txt`.

2. В настройках проекта (Variables) добавь переменную:

   ```
   BOT_TOKEN=твой_токен_от_BotFather
   ```

3. **Важно про базу данных.** Бот использует SQLite-файл (`bot.db`) прямо на диске
   контейнера. На Railway файловая система по умолчанию **не персистентная** —
   при каждом передеплое диск может быть пересоздан, и `bot.db` со всеми
   пользователями и балансами исчезнет.

   Чтобы этого избежать, подключи **Volume**:
   - В Railway: Project → New → Volume, примонтируй его, например, на `/data`.
   - Добавь переменную окружения `DB_PATH=/data/bot.db`.
   - Бот уже читает `DB_PATH` из окружения (см. `config.py`), так что больше
     ничего менять не нужно.

   Без volume бот будет исправно работать, но данные будут теряться при каждом
   редеплое — для теста это ок, для боевой эксплуатации обязательно нужен volume.

4. Задеплой — Railway запустит `python bot.py` как worker-процесс (поллинг,
   без вебхуков, отдельный веб-сервер не нужен).

## Что стоит доработать перед боевым использованием

- **Лимит заданий в час** на пользователя — сейчас его нет, стоит добавить,
  чтобы защититься от накрутки через множество каналов подряд.
- **Приватные каналы без username** — сейчас ссылка для перехода строится
  по шаблону `t.me/c/...`, который работает не для всех клиентов; для приватных
  каналов лучше генерировать реальную invite-ссылку через `create_chat_invite_link`.
- **Капча/анти-бот на `/start`**, если заметите наплыв автоматических аккаунтов.
- **Логи и админ-панель** — хотя бы простая команда для владельца бота, чтобы
  смотреть статистику (сколько пользователей, каналов, подписок в день).
